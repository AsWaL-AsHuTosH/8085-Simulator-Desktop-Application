COMPONENTS

  1. PROGRAM LOADER PANEL

    1. It display the current location from where the program will be loaded.
    2. Loading location can be changed by setting a new loading location.

      Note: In order to change program loading location, provide new address in the textbox asking for loading address
      and press SET button.

  2. FLAG REGISTER PANEL

    It display the status of Flag Register.


  3. REGISTER PANEL

    It display the content of A, B, C, D, E, H & L Register in hexadecimal.


  4. MEMORY INITIALIZER PANEL


    It provide way to feed data in memory location ranging from [0000 - FFFF].

    Direction To Use: Provide the address and data in the corresponding text box and click on SET button.


  5. MEMORY VIEWER PANEL

    It provide way to view the data in the memory.

    Direction To Use: Provide the starting address and the count(number of locations to be accessed from the starting address) in the corresponding text box and click on the VIEW button.


  6. DEBUGGER PANEL

    It provide mechanism to debug the code.

    Direction To Use
    1. Write the code.
    2. Make sure to save the file.
    3. Hit Debug button which will enable the Execute and STOP button.
    4. Hit the execute button to execute/run the line number shown in the Debugger Panel.
    5. Hit the STOP button to stop debugging.

    Note:
    • Debugger will stop automatically after executing HLT instruction.
    • Use STOP button to stop debugging in between.


KEY POINTS

  1. Loop/Jump/Call


    In order to jump/call to a specific line of code create a label.
    Label is any string(alpha-numberic) ending with ':', for example: "LOOP:", "HERE:" etc.

    Note: Label should not match any Reserve Keyword in 8085 i.e., "JNZ:", "MOV:", "mvi:", "A:", "Call:", etc., All are be invalid labels

    Example of a program using label

     LDA 2050
    MOV D, A
    MVI B, 0A
    LXI H, 3050
    MVI C, 00
    LOOP: MOV A, D ADD C
    MOV M, A
    INX H
    MOV C, A
    DCR B
    JNZ LOOP
    HLT


  2. How to use comments in code

    Any line starting with '@' symbol will be considered as comment in the code.

    Example of a program using Comments

     MVI A, 00
    MVI B, 11
    @Next line will add B register content to A and will store result in A
    ADD B
    HLT

    Note: You can not comment in between, i.e.,

    • MOV A, B @Will copy the content of B to A   [WRONG]
    • @Next line will copy the content of B to A   [CORRECT]
      MOV A, B


  3. Empty lines

    All empty lines in between the codes will be ignored.


  4. Hexadecimal Values

    Do not use H or h to represent the hexadecimal values as application is designed in such a way that it will always consider every immediate data in hexadecimal number system only.
    • MVI A, 3F  [CORRECT]
    • LXI H, 1F33  [CORRECT]
    • MVI A, 3Fh  [WRONG]
    • LXI B, 1F33H  [WRONG]

Note: While using PCHL the application will through an erro if:
  1. There exist no instruction at the location pointed by HL.

  2. The Location pointed by HL is not the starting location of a instruction, for eg:
    Let us consider a program given below

     MVI H, 08H
    MVI L, 01H
    PCHL
    HLT

    If the loading location is 0800 the program in memory will be as follows:

    ADDRESS OPCODE
    0800 26 [MVI_H]
    0801 08 [DATA]
    0802 2E [MVI_L]
    0803 01 [DATA]
    0804 E9 [PCHL]
    0805 76 [HLT]
    In this case, after executing PCHL, PC will point or store 0801 and Jump to that location, but at 0801 there is not valid instruction because 8081 is containing Data, Thats why the application will through error in this case. In above program the valid locations at which we can shift the PC are 0800, 08002, 0804, 0805. So be careful while using PCHL, HL should provide a valid location to PC.
