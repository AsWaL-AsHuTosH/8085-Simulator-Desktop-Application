COMPONENTS

  1. FLAG REGISTER PANEL

    2. It will display the status of Flag Register.

  2. REGISTER PANEL

    3. It display the content of A, B, C, D, E, H & L Register in hexadecimal.

  3. MEMORY INITIALIZER PANEL

    4. It provide way to feed data in memory location ranging from [0000 - FFFF].

    Direction To Use: Provide the address and data in the corresponding text box and click on SET button.

  4. MEMORY VIEWER PANEL

    5. It provide way to view the data in the memory.

    Direction To Use: Provide the starting address and the count(number of locations to be accessed from the starting address) in the corresponding text box and click on the VIEW button.

  5. DEBUGGER PANEL

    6. It provide mechanism to debug the code.

    Direction To Use
    1. Write the code.
    2. Make sure to save the file.
    3. Hit Debug button which will enable the Execute and STOP button.
    4. Hit the execute button to execute/run the line number shown in the Debugger Panel.
    5. Hit the STOP button to stop debugging.

    Note:
    • Debugger will stop automatically after executing HLT instruction.
    • Use STOP button to stop debugging in between.


KEY POINTS

  1. Loop/Jump


    2. In order to jump to a specific line of code create a Jump-Point. Jump Point is any string ending with ':'.
    for example: "LOOP:", "HERE:" etc.

    Note: All Jump-Point strings will be considered in their uppercase form i.e., "LOOP:", "Loop:", "loop:", "loOP:", . . . . . . will be considered as same Jump-Point as all of them same uppercase form "LOOP:".

    Example of a program using Jump-Point

    LDA 2050
    MOV D, A
    MVI B, 0A
    LXI H, 3050
    MVI C, 00
    LOOP: MOV A, D ADD C
    MOV M, A
    INX H
    MOV C, A
    DCR B
    JNZ LOOP
    HLT

  2. How to use comments in code

    3. Any line starting with '@' symbol will be considered as comment in the code.

    Example of a program using Comments

    MVI A, 00
    MVI B, 11
    @Next line will add B register content to A and will store result in A
    ADD B
    HLT

    Note: You can not comment in between, i.e.,

    • MOV A, B @Will copy the content of B to A   [WRONG]

    • @Next line will copy the content of B to A   [CORRECT]
      MOV A, B

  3. Empty lines

    4. All empty lines in between the codes will be ignored.

  4. Hexadecimal Values

    5. Do not use H or h to represent the hexadecimal values as application is designed in such a way that it will always consider every immediate data in hexadecimal number system only.

  5. How to write a avoid syntax error

    6. Separate every token by a space, i.e.,

    • MOV A,B       [WRONG]
    • MOV A, B        [CORRECT]
    • MOV A , B       [CORRECT]
    • MOV A ,B        [CORRECT]
    • LOOP:MVI A, 00   [WRONG]
    • LOOP: MVI A, 00  [CORRECT]
